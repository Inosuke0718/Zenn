React はモダンな Web フロントエンド開発において、最も人気のあるライブラリの一つです。しかし、初学者にとっては「コンポーネント」「 JSX 」「 State 」など、特有の概念が多くて戸惑うことも多いのではないでしょうか。

「何から手をつければいいの？」「 Props と State の違いがわからない……」そんな風に迷うこと、ありませんか？

この記事では、 React の基本概念から環境構築、そして実践的なコンポーネントの作成と状態管理（ State ）までを、現役エンジニアの視点でわかりやすくステップバイステップで解説します！

## 🚀 1. Reactとは？なぜ使われるのか？

React は、複雑な UI （ユーザーインターフェース）を ** 効率よく構築する ** ために開発された JavaScript ライブラリです。主に以下の 3 つの強力な特徴を持っています。

### 🧩 コンポーネント指向

画面を構成する要素（ボタン、ヘッダー、カードなど）を「 ** レゴブロック ** 」のように細かく分割し、それらを組み合わせて Web アプリを構築します。これによりコードの再利用性が高まり、メンテナンスがとても楽になります。

### ⚡ 仮想DOM（Virtual DOM）による高速な描画

データに変更があった際、ページ全体を再読み込みするのではなく、変更された部分だけを ** 仮想DOM ** を使って計算し、実際の画面に反映させます。これにより、非常に高速でスムーズなユーザー体験を実現します。

### 📱 React Nativeへの応用

React の概念を学べば、同じ原則を使って iOS や Android のネイティブアプリも開発できるようになります。「一石二鳥」のスキルと言えますね！

## 🛠️ 2. 環境構築：最初のReactアプリを作ろう

React アプリの開発を始めるための準備は非常にシンプルです。事前に ** Node.js ** がインストールされていることを確認してください。

ターミナルを開き、以下のコマンドを実行するだけで、開発に必要なファイル群（ボイラープレート）が自動でセットアップされます。

```bash
# アプリの作成（"my-app"は任意のプロジェクト名）
npx create-react-app my-app

# 作成したディレクトリに移動してサーバーを起動
cd my-app
npm start
```

これでローカルサーバーが立ち上がり、ブラウザで React の初期画面が表示されます。最初の一歩、おめでとうございます！

## 📁 3. フォルダ構造の理解

自動生成されたプロジェクトの中身を見ると、いくつかのフォルダとファイルが存在します。開発を進める上で特に重要なものを把握しておきましょう。

| フォルダ・ファイル | 役割                                                                                           |
| ------------------ | ---------------------------------------------------------------------------------------------- |
| `node_modules/`    | プロジェクトを動かすためのライブラリがすべて詰まっています。基本触りません。                   |
| `public/`          | 画像やフォントなど、直接配信される静的ファイルを配置します。                                   |
| `index.html`       | ブラウザが最初に読み込むファイルです。 `<div id="root"></div>` に React の UI が注入されます。 |
| `src/`             | ** 開発の中心となる「心臓部」 ** です。私たちがコードを書くのはここです！                      |
| `App.js`           | アプリ全体の起点となるメインのコンポーネントです。                                             |

## ✨ 4. JSXの魔法：JavaScriptの中にHTMLを書く

`App.js` を開くと、 JavaScript のファイルの中に直接 HTML のようなタグが書かれていることに気づくでしょう。これは ** JSX (JavaScript XML) ** と呼ばれる構文です。

JSX の最大の特徴は、 ** UIの構造（HTML）と動的な処理（JavaScript）を一つのファイルにまとめて書ける ** ことです。

```jsx
// JSXの基本的な書き方
const name = "React初心者";

function App() {
  return (
    <div className="container">
      <h1>こんにちは、 {name} さん！</h1>
    </div>
  );
}
```

- ** ポイント ** : 波括弧 `{}` を使うことで、 JSX の中に JavaScript の変数や式を直接埋め込むことができます。また、 CSS クラスを指定する際は `class` ではなく `className` を使用するルールがあります。

## 🏹 5. コンポーネントとProps：再利用可能なUIを作る

React では、 UI のパーツを ** 関数コンポーネント ** として定義します。大文字から始まる関数を作り、 JSX を `return` するだけです。

### コンポーネントの基本

```jsx
// src/components/Button.jsx
export function Button() {
  return <button className="btn">クリックしてね</button>;
}
```

### Propsでデータを渡す

同じ見た目のカードコンポーネントを複数作りたいけれど、それぞれ表示する「名前」や「説明」を変えたい場合があります。このような時に使うのが ** Props（プロップス） ** です。

Props は、親コンポーネントから子コンポーネントへデータを渡すための仕組みです。

```jsx
// 子コンポーネント：Propsを受け取って表示する
function Card(props) {
  return (
    <div className="card">
      <h2> {props.title} </h2>
      <p> {props.description} </p>
    </div>
  );
}

// 親コンポーネント：Propsにデータを渡して呼び出す
function App() {
  return (
    <div className="card-container">
      <Card
        title="Reactの基本"
        description="コンポーネントについて学びましょう。"
      />
      <Card title="Propsの使い方" description="データを動的に渡す方法です。" />
    </div>
  );
}
```

データが多い場合は、オブジェクトにまとめてスプレッド構文（ `{...data}` ）で一気に渡すテクニックもよく使われます。

## 💾 6. State（状態）の管理：アプリに記憶力を持たせる

Props が「外部から渡される固定データ」だとすれば、 ** State ** は「コンポーネントが自ら保持し、変化するデータ（記憶）」です。

例えば、「いいね」ボタンのクリック数など、ユーザーの操作によって値が変わる UI を作るには `useState` フックを使用します。

```jsx
import { useState } from "react";

function Counter() {
  // count: 現在の状態（初期値0）
  // setCount: 状態を更新するための関数
  const [count, setCount] = useState(0);

  return (
    <div className="counter">
      <p>現在のカウント: {count} </p>
      {/* ボタンをクリックすると、setCountが呼ばれてcountが増える */}
      <button onClick={() => setCount(count + 1)}>+1 増やす</button>
      <button onClick={() => setCount(count - 1)}>-1 減らす</button>
    </div>
  );
}
```

`setCount` が実行されて State が更新されると、 React は自動的にそのコンポーネントを再描画し、画面の表示を最新の状態にアップデートしてくれます。賢いですね！

### 条件付きレンダリング

State の値に応じて表示を切り替えることも簡単です。 JSX 内では ** 三項演算子 ** （ `条件 ? 真の場合 : 偽の場合` ）を使うとスッキリ記述できます。

```jsx
// isLoginがtrueなら「ログアウト」、falseなら「ログイン」を表示
<button>{isLogin ? "ログアウト" : "ログイン"}</button>
```

## 🏁 まとめ

React の学習は以下のステップで進めるとスムーズです！

1. ** JSX ** で画面の構造を作る感覚を掴む
2. UI を意味のある単位で ** コンポーネント ** に分割する
3. ** Props ** を使ってコンポーネント間でデータを受け渡す
4. ** State (useState) ** を使って、ユーザーの操作に応じた動的な変化を実装する
