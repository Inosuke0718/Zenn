最近話題の **MCP (Model Context Protocol)**、皆さんはもう使っていますか？

![mcp](https://github.com/Inosuke0718/Zenn/blob/main/images/MCP.png?raw=true)
_出典：[クラウドエース](https://cloud-ace.jp/column/detail519/)_

Claude DesktopやCursor、そして最近登場したClaude Codeなど、MCP対応のAIツールが増える中で、「結局どのサーバーを入れればいいの？」と迷っている方も多いはず。

この記事では、最近の注目ツール（Draw.ioなど）から、開発効率を爆上げする鉄板サーバーまで、**エンジニアにおすすめのMCPサーバー**を厳選して紹介します。

:::message
この記事は2026年2月時点の情報を基にしています。
:::

# 1. 最近話題のMCPサーバー 4選

まずは、直近で注目を集めている最新ツールから紹介します。

## Draw.io MCP（おすすめ度：★★★★★）

**「AIに指示するだけで、システム構成図を作りたい」**

そんな願いを叶えてくれるのが,Draw.io公式のMCPサーバーです。

- **特徴**: テキストからXML、CSV、Mermaid形式で図を生成可能。
- **ユースケース**:
  - 「AWSのサーバーレス構成図を描いて」と指示して、編集可能なXMLを出力
  - 複雑なフローチャートのドラフト作成
- **導入**: `npx @drawio/mcp` で一発インストール

詳細なレイアウト調整も可能で、現時点での作図系MCPとしては頭一つ抜けています。

## Excalidraw MCP

手書き風の図が書けるExcalidrawのMCP版。
「手書き感」が好きならアリですが、導入が少し手間（リポジトリのCloneが必要など）なのが現状のネック。リンク共有機能などは便利です。

## Obsidian CLI & Happy

- **Obsidian CLI**: Obsidianユーザー待望のツール。ターミナルから直接ノート作成や検索が可能（※現在はCatalystライセンス限定機能が含まれる場合あり）。
- **Happy**: スマホからClaude Codeを操作できるモバイルクライアント。外出先での「Vibe Coding（雰囲気コーディング）」に最適。

## Serena MCP

**「コードベース全体を深く理解する」**ためのエージェント向けツールキット。

- **特徴**: 単なるgrep検索ではなく、コードの意味的（セマンティック）な検索や、シンボルの参照関係を理解したナビゲーションが可能。
- **メリット**: 大規模プロジェクトで「この関数どこで使われてる？」や「この変更の影響範囲は？」といった質問に対して、非常に精度の高い回答が得られます。リファクタリングのお供に。

# 2. エンジニア必須！鉄板MCPサーバー 5選

ここからは、**「これを入れておけば開発効率が変わる」** という実用重視のサーバーです。

## ① GitHub MCP Server

もはや説明不要の必須ツール。

- **できること**:
  - リポジトリの検索・ファイル閲覧
  - Issueの作成・確認
  - PRの内容チェック
- **メリット**: ブラウザを行き来せず、エディタやAIチャット内だけで「あのリポジトリの`config`どうなってたっけ？」が完結します。

## ② Docker MCP Server

コンテナ周りのトラブルシューティングが爆速になります。

- **できること**: コンテナ一覧表示、ログ確認、コンテナ操作
- **メリット**: エラーが出た時、AIに「**Dockerのログを見て原因教えて**」と投げるだけで、ログ取得→分析まで自律的にやってくれます。

## ③ PostgreSQL / SQLite MCP Server

DBクライアントを開くのが面倒なあなたに。

- **できること**: スキーマ確認、読み取り専用クエリの実行
- **メリット**: 自然言語でDB分析ができます。「usersテーブルから、先週登録した人の数を数えて」といった指示でSQLを書いて実行してくれます。

## ④ Chrome DevTools MCP

ブラウザのデバッグもAIにお任せ。

- **できること**: Consoleログの監視、ネットワークリクエストの分析、Lighthouse監査の実行。
- **ユースケース**:
  - 「コンソールに出てるエラーの原因を特定して修正案を出して」
  - 「ページの読み込みが遅い原因をネットワークタブから分析して」

## ⑤ Playwright MCP

ブラウザ操作の自動化・テスト作成に。

- **特徴**: AIがブラウザ（DOM）を直接操作。クリック、入力、スクリーンショット撮影などが可能。
- **ユースケース**:
  - 「このサイトのログインフローのE2Eテストコードを書いて」
  - 「サイトを巡回してデザイン崩れがないかチェックして」

# 3. リサーチ＆情報収集系

AIの知識は学習データ（カットオフ）に依存しますが、これを補完するのが検索系MCPです。

## Brave Search MCP

プライバシー重視の検索エンジンBraveのAPIを利用。
高速かつ低コストで、AIに最新ニュースやドキュメントを検索させることができます。

## Tavily / Exa

**「AIのための検索エンジン」** です。
通常のGoogle検索と違い、**LLMが読み取りやすい形式**でデータを返してくれるため,回答の精度が段違いに良くなります。Deep Research（詳細調査）を行わせたい場合はこちらがおすすめ。

# まとめ：おすすめの構成例

個人的な推奨構成は以下の通りです。

:::message box
**とりあえずこれ入れておけセット**

1. **GitHub** (コード管理)
2. **Filesystem** (ローカルファイル操作・必須)
3. **Brave Search** or **Tavily** (最新情報検索)
4. **Draw.io** (ドキュメント作成用)
5. **Serena** (コードベース深層理解)
   :::
