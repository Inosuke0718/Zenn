Google の Gemini Advanced（Proプラン）に加入したなら、ブラウザ版だけでなくターミナルから直接 AI を呼び出せる **Gemini CLI** の活用が非常におすすめです。

![Gemini CLI Header](https://raw.githubusercontent.com/Inosuke0718/Zenn/main/images/gemini-cli-header.png)

「ターミナルから AI を使うのは少し難しそう...」と思っていませんか？ 実は、設定次第でブラウザ版よりも遥かに強力な、自分専用の **最強コーディングパートナー** に進化させることができるんです！

この記事では、Windows 環境に Gemini CLI を導入する手順から、精度をグンと上げるおすすめの初期設定、そして強力な「 YOLO モード」まで徹底解説します。

> [!TIP]
> **Antigravity ユーザーへの耳寄り情報**
> 話題の AI エディタ「 Antigravity 」に組み込まれている Gemini 3.1 Pro の利用枠と、この Gemini CLI の利用枠（クォータ）は **完全に独立** しています。
> Antigravity で重い処理を回して制限に引っかかっても、CLI 側は別枠で 1 日 1500 回（RPM120）までサクサク動きます。用途を分けて使いこなすのが、賢い AI 運用のコツですよ！

## 🚀 1. 導入：Node.js と CLI のインストール

Gemini CLI は **Node.js** 上で動作します。まずは環境を確認しましょう。

### 🛠️ インストール手順

まだ Node.js が入っていない方は、 [公式サイト](https://nodejs.org/) からインストールしておきましょう（v20 以上を推奨します）。

環境が整ったら、 PowerShell を開き、以下のコマンドを実行するだけです。

```powershell
npm install -g @google/gemini-cli
```

インストール後、 `gemini` コマンドで起動します。初回はテーマ選択や Google アカウントでの認証（ブラウザが立ち上がります）を求められるので、画面の指示に従って済ませれば準備完了です！

## ⚙️ 2. 必須の初期設定：モデル固定と精度の向上

プログラミングで活用するなら、デフォルト設定のままでは AI が少し「おしゃべり」すぎたり、毎回モデルを指定するのが面倒だったりします。
以下のコマンドで、 CLI を **コーディング特化のストイックな設定** にカスタマイズしましょう！

### ① モデルを「 Gemini 3.1 Pro 」に固定する

複雑なロジック構築やリファクタリングを任せるため、デフォルトのモデルを Pro に固定します。

```powershell
gemini config set model gemini-3.1-pro
```

### ② Temperature（温度）を下げて正確性を上げる

AI の「創造性」を少し抑えることで、論理的で正確なコード出力を優先させます。 **0.2** くらいがコーディングにはベストバランスだと私は思っています。（異論はもちろん認めます。）

```powershell
gemini config set temperature 0.2
```

### ③ システムプロンプト（基本姿勢）の登録

毎回「コードだけ出して」「日本語で」と指示するのは面倒ですよね。あらかじめ基本姿勢を教えておきましょう。

```powershell
gemini config set system-prompt "あなたは優秀なソフトウェアエンジニアです。回答は必ず日本語で行ってください。コードを提示する際は、冗長な前置きや解説は最小限にし、そのまま実行・組み込み可能なコードブロックを優先して出力してください。"
```

## 📝 3. 実践テクニック：プロジェクト固有のルール設定

プロジェクトごとに「守ってほしいコーディング規約」や「前提知識」ってありますよね。 Gemini CLI では、これをファイルで管理できます。

プロジェクトのルートディレクトリに `GEMINI.md` というファイルを作成し、ルールを記述しておきましょう。

**`GEMINI.md` の記述例:**

```markdown
# プロジェクト基本ルール

- 言語: Java 21, Python 3.12
- 命名規則: クラス名は PascalCase、メソッド・変数は camelCase を厳守。
- コメント: Javadoc形式で、複雑なロジックには必ず日本語で意図を記載すること。
```

この `GEMINI.md` は、 Antigravity などの他の Google 系 AI ツールとディレクトリ構造を共有しているため、 **一度書いておけばどのツールから開いても同じルールで AI が動いてくれる** のが最大のメリットです！

## ⚡ 4. 上級者向け：「 YOLO モード」で全自動化

Gemini CLI の真骨頂とも言えるのが **YOLO（You Only Live Once）モード** です。

通常、 AI がファイルの書き換えやコマンドを実行する前には、ユーザーの承認（Yes/No）が求められます。しかし、 YOLO モードを有効にすると、 **この確認をすべてスキップし、 AI が全自動でタスクを完遂するまで走り続けます。**

### 🏃 YOLO モードでの起動方法

```powershell
gemini --yolo
```

対話モード起動中であれば、 `Ctrl + Y` を押すことでいつでもオン / オフの切り替えが可能です。

> [!CAUTION]
> **取り扱い注意！**
> リントエラーの一括修正や定型的な環境構築を一瞬で終わらせる強力な機能ですが、 AI が意図しないファイルを削除・変更してしまうリスクもあります。 YOLO モードを実行する前は、 **必ず Git などで変更をコミット（保存）してから** 実行するようにしましょう。

## 🏁 まとめ

Gemini CLI は、ブラウザを開く手間なく、普段作業しているターミナルからシームレスに高度な AI を呼び出せる強力なツールです。

最後に、これだけは覚えておきたいポイントをまとめました。

- **利用枠の賢い使い分け** : Antigravity と CLI のクォータは別枠（独立）。
- **ストイックな設定** : モデルを Pro に固定し、温度を下げるのが鉄則。
- **GEMINI.md の活用** : プロジェクトの規約を共有して「分かっている AI 」にする。
- **YOLOモードのパワー** : 全自動実行。ただし Git コミットは必須。

利用状況（消費トークンやクォータ残量）が気になった時は、対話モード中に `/stats` コマンドでいつでも確認できます。
