# 【脱黒い画面】VS Code × GitLensで実現する、一番やさしいGitHub操作入門 🚀

## はじめに：GitHubは「怖くない」

「ターミナルでコマンドを叩くのが怖い」
「コンクリフト（競合）が起きそうで不安」
「今、自分が何をしているかわからなくなる」

そんな "GitHubアレルギー" を持っている初心者の方へ。

実は、VS Code と拡張機能 **「GitLens」** を使えば、GitHubの操作はスマホの写真管理くらい直感的になります。黒い画面（ターミナル）を開く必要は一切ありません。

この記事では、「コマンドを一切打たずに」 安全にGitHubを操作し、開発を管理する方法を解説します。

## 1. 準備：最強の相棒「GitLens」を入れる

まずはVS Codeに拡張機能を入れましょう。これがあるだけで、目に見えないGitの履歴が「見える化」されます。

- **拡張機能名**: GitLens — Git supercharged
- **インストール方法**: VS Code左側の拡張機能タブ（四角いブロックのアイコン）で「GitLens」と検索してInstall

:::message
**💡 ここがポイント**
インストールすると、エディタ上のコードの横に薄い文字で「誰が・いつ・この行を書いたか」が表示されるようになります（インラインBlame機能）。
「これ、誰が書いたコード？」が1秒で分かるようになり、開発の安心感が段違いになります。
:::

## 2. 基本の操作：変更を確認する（ステージング）

コードを書いたら、いきなりコミットするのではなく、「何を変えたか」を確認するのが上級者への第一歩です。

1. VS Code左側の **「ソース管理アイコン（枝のようなマーク）」** をクリック。
2. 「変更」欄にあるファイル名をクリック。

**GitLensのパワー発動：**

- 左側に「変更前」、右側に「変更後」が並んで表示されます。
- 行単位でハイライトされるので、余計な空白や改行が入っていないか一瞬で分かります。

確認してOKなら、ファイル名にマウスを乗せた時に出る **+ ボタン（変更をステージ）** を押します。
これが `git add` コマンドと同じ操作です。「この変更を記録するぞ」という荷造りの合図です。

## 3. 歴史を残す：コミットする

ステージング（荷造り）ができたら、メッセージを付けて記録を確定させます。

1. ソース管理パネルの上部にある入力欄に、「ログイン機能を追加」などのメッセージを入力。
2. **「コミット」ボタン（青いボタン）** をクリック。

これで `git commit` 完了です。簡単ですね！

## 4. 実験用の部屋を作る：ブランチ操作

開発に慣れてくると「新しい機能を試したいけど、今の動いているコードは壊したくない」という場面が出てきます。そんな時は **「ブランチ（並行世界）」** を作りましょう。

### 新しいブランチを作る

1. VS Codeの画面左下のステータスバーにある **「main」（または master）**という文字をクリック。
2. 画面上部に出てくるメニューから **「+ 新しいブランチの作成...」** を選択。
3. 好きな名前を入力します（例：`feature/test-design`）そしてEnter。

これだけで、安全な実験用の部屋（ブランチ）に切り替わりました。ここでいくらコードを書き換えても、元の `main` ブランチには影響しません。

### ブランチを切り替える

「やっぱり元のコードに戻りたい」と思ったら：

1. 左下のブランチ名（先ほど作った名前になっています）をクリック。
2. リストから `main` を選ぶ。

これだけで、ファイルの中身が瞬時に「ブランチを作る前の状態」に戻ります。まるでタイムマシンのように、行ったり来たりできます。

## 5. 統合する：マージ（Merge）

実験用ブランチで書いたコードがうまく動いたら、メインのコード（`main`）に取り込みましょう。これを「マージ」と呼びます。

手順は「移動して、取り込む」の2ステップです。

### 移動する:

まず、取り込み先である `main` ブランチに切り替えます（左下のステータスバーから `main` を選択）。

### 取り込む:

1. VS Code左側の「ソース管理アイコン」をクリック。
2. 「ソース管理」というタイトルの横にある「…」メニューをクリック。
3. **「ブランチ」 > 「ブランチをマージ...」** を選択。
4. リストから、取り込みたいブランチ（例：`feature/test-design`）を選びます。

これで、実験用ブランチの内容が `main` に合体しました！

:::message
**💡 もし「競合（コンクリフト）」したら？**
焦らなくて大丈夫です。VS Codeが「どっちのコードを採用する？」と見やすいボタン付きで聞いてくれます。「現在の変更を取り込む」などのボタンを押すだけで解決できます。
:::

## 6. 世界に保存：プッシュする

最後に、自分のPCで記録した内容をGitHub（クラウド）に送信します。

1. ソース管理パネルにある **「変更の同期（またはプッシュ）」** ボタンをクリック。

これだけです。

**GitLensを使っていると…**
ステータスバーの下部に `↑1` のような矢印が出ます。これは「GitHubに送っていない履歴が1つあるよ」という合図です。ここを見るだけで「プッシュ忘れ」を防ぐことができます。

## 7. GitLensの真骨頂：「過去へのタイムトラベル」

初心者が一番助かるのが、GitLensの **「File History」** 機能です。

「あ、昨日のコードの方が良かったかも…」
「いつの間にかバグってる…いつ変更したんだっけ？」

そんな時は、GitLensのサイドバー（またはファイル右上の時計アイコン）から **File History** を開きます。
そのファイルが過去どのような変化を辿ってきたかが、リストで一覧表示されます。クリックするだけで、その時点のコードにジャンプして確認できます。

## おわりに：見えると、開発はもっと楽しくなる

GitLensで「履歴」と「現状」が見えるようになると、GitHub操作の恐怖心は消え、むしろ「ログを残す楽しさ」に変わります。

最初は **「変更を見る → +ボタンを押す → コミットする」** という基本の流れだけで十分です。慣れてきたらブランチ機能を試してみてください。

ぜひVS Code × GitLensの組み合わせで、快適なZenn執筆・開発ライフを始めてみてください！
