# 【初学者向け】Spring Bootでポートフォリオを作って公開するまで

この記事では、 **Spring Boot** を使って「人に見せられるポートフォリオ」を作り、インターネット上に公開し、GitHubのREADMEを整えるまでの全体像をガイドします。

ゴールは「完成したアプリのURLを採用担当や友人に共有できる状態にすること」です。

---

## このガイドの進め方（重要）

1.  最初に **MVP版（必要最小限の機能だけを持ったバージョン）** を作ります。
2.  まずは動く状態にし、その後で見た目や機能を追加していきます。
3.  途中で詰まったら、遠慮なくAI（ChatGPTやGeminiなど）に相談しましょう！

> **コツ：**
> 最初から「あれもこれも」と全部入りの完璧なアプリを目指すと、途中で挫折しやすくなります。
> 実際の現場でも、まずは「動く最小の形」を完成させることが非常に重要視されています。

---

## 1. アプリのテーマ決め（何でもOK）

作るテーマは何でも大丈夫です。迷った場合は、以下の3パターンから選ぶのがおすすめです。

- **自分が今、困っていることや欲しいもの** （例：日々の学習記録アプリ、シンプルなタスク管理アプリ）
- **自分の趣味や好きなこと** （例：スポーツの練習記録、読んだ漫画の感想メモ、筋トレの記録）
- **既存サービスの簡易版（クローン）** （例：シンプルなTODOアプリ、簡易的なフリマアプリ、掲示板アプリ）

## 2. 要件の決定（AIと一緒に詰める）

何を作るか決めたら、次は「どんな機能が必要か」を考えます。難しく考える必要はありません。ここでAIの力を借りましょう。

### AIに投げるプロンプト例

以下の文章をコピーして、あなたの選んだテーマに合わせて編集し、AIに投げてみてください。

```text
あなたは経験豊富なプロダクトマネージャーです。
私はプログラミング初学者で、Spring Bootを使って「〇〇（あなたのテーマ）アプリ」を作りたいと考えています。

初学者がAIを活用しながら、約1週間で完成させられる範囲の「MVP要件（必要最小限の機能）」を、箇条書きで具体的に提案してください。
また、MVPが完成した後に段階的を追って追加できる、魅力的な拡張アイデアもいくつか提案してください。
```

## 5. MVPを実装する（ここが本編）

さあ、いよいよコードを書いていきます！現代の開発では、AIを搭載したコードエディタ（AI IDE）を使うのが圧倒的に効率的です。

おすすめのAI IDEとその特徴をまとめました。

| ツール名    | 公式サイト                      | 特徴・おすすめポイント                                                             | 料金・無料枠                         |
| ----------- | ------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------ |
| Cursor      | cursor.com cursor​              | AI IDEの先駆け。AI IDEといったらこれ                                               | 有料プラン（Pro）は月額20ドル。      |
| Antigravity | antigravity.google antigravity​ | Googleが満を持して作ったAI IDE。無料でOpus4.6が使えるのはうれしい！                | 無料でOpus4.6が使える。              |
| KIRO        | kiro.dev kiro​                  | 無料のボーナス枠が非常に大きいです。VIBEとSPEC駆動開発モードが選べるのがうれしい。 | Sonet4.5がたくさん使える無料枠がいい |

### AI開発のアプローチ：VIBEコーディング vs SPEC駆動開発

AIを使った開発には、大きく分けて2つのアプローチがあります。

- **VIBEコーディング:** AIと対話しながら、雰囲気（Vibe）でどんどんコードを生成していくスタイル。圧倒的にスピードが速いのが特徴です。
- **SPEC駆動開発:** 最初にある程度仕様（Spec）を定義し、それに沿ってAIにコードを書かせるスタイル。意図した通りの動きになりやすく、コードの構造も理解しやすいです。

個人的には、 **学習も兼ねて開発を行う初学者には「SPEC駆動開発」をおすすめします。** 自分が何を作ろうとしているのかを整理しながら進められるため、プログラミングの理解が深まります。とにかく早く形にしたい！という場合はVIBEコーディングでも良いでしょう。

## 6. デプロイ（公開URLを作る）

ローカルで動くようになったら、いよいよインターネット上に公開（デプロイ）します。デプロイしてURLを発行して初めて「人に見せられるポートフォリオ」になります。

### どこにデプロイする？

- **Render:** 無料枠があり、GitHubと連携して簡単にデプロイできるため、初学者に最もおすすめです。（※無料枠の場合、一定時間アクセスがないとスリープ状態になり、次のアクセス時に起動に少し時間がかかるという特徴があります。）
- **Heroku:** 以前は定番でしたが、現在は無料枠が廃止されています。公式のドキュメントは豊富です。

RenderやHerokuでのデプロイ方法は、検索すると多くの解説記事が見つかります。

## 7. GitHubのREADMEを書く（ポートフォリオの顔）

最後に、GitHubのREADMEをしっかりと書きましょう。採用担当者も見る重要な「顔」になります。

READMEが何も書かれていないと、「どんなアプリなのか」「どうやって動かすのか」が全く伝わりません。

※もしデプロイしないのであれば、紹介動画や画像やGif動画もREADMEに入れ込んで、なるべくわかりやすい構成にしましょう

### READMEの基本構成

以下の内容を盛り込むのが定番です。

- **アプリの目的・概要** （何ができるアプリか）
- **デモのURL** （実際に触れるURL）
- **使用した技術** （Java、Spring Bootのバージョンなど）
- **主要な機能**
- **ローカル環境での動かし方** （Setup）
- **アプリのスクリーンショット** （数枚あると一気にイメージが伝わります）

### READMEテンプレート

````md
# あなたのアプリ名

## Overview

Spring Bootを使って作成した〇〇アプリです。
（アプリの簡単な説明を1〜2文で書く）

## Demo

- URL: [ここにデプロイ先のURLを貼る]

## Features

- 〇〇の登録・編集機能
- 〇〇の一覧表示機能
- ログイン・ログアウト機能

## Tech Stack

- Java (バージョン)
- Spring Boot (バージョン)
- データベース (例: PostgreSQL, MySQL, H2 Database)
- フロントエンド (例: Thymeleaf, React, Vue.jsなど)

## Setup

ローカル環境で動かすための手順です。

1. リポジトリをクローン

```bash
git clone [リポジトリのURL]
```
````

2. プロジェクトのディレクトリに移動

```bash
cd [プロジェクトフォルダ名]
```

3. アプリケーションの起動

```bash
./mvnw spring-boot:run
```

## Screenshots

（ここにアプリの操作画面のスクリーンショット画像を数枚貼る）

## License

MIT

