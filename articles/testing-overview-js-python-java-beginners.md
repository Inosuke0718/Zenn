## はじめに

「E2Eテストって何？」「Vitest って React 専用じゃないの？」「Python や Java のテストって何を使えばいいの？」

テストについて学び始めると、次から次へと疑問が湧いてきますよね。

この記事では、そんなテストの全体像を「テストピラミッド」という考え方から整理し、JavaScript・Python・Java それぞれの定番ツールまで、初学者向けにサクッと解説します！

## テストの役割分担を「テストピラミッド」で理解しよう

まず、テストには大きく3つの層があることを理解しましょう。これを図にしたのが「テストピラミッド」です。

![テストピラミッド](https://raw.githubusercontent.com/your-username/your-repo/main/articles/testpiramid.png)

| 層 | テストの種類 | 何をテストする？ | 特徴 |
| :--- | :--- | :--- | :--- |
| **上層** | **E2Eテスト** | ユーザーの操作を一気通貫で確認 | 遅くてコストが高いが、全体の動作を保証できる |
| **中層** | **APIテスト** | APIの仕様（リクエストとレスポンス）を確認 | E2Eより速く、サーバー側のロジックを検証 |
| **下層** | **単体(ユニット)テスト** | 関数やコンポーネント単体の動作を確認 | **高速で低コスト。** たくさん書くべき |

ピラミッドの**下に行くほど「速く、安く、たくさん書く」**、**上に行くほど「遅く、高く、少なく書く」**のが基本戦略です。

## どのテストから始めるべき？ →「下から」が鉄則！

テストを自動実行（CI/CD）する際は、**「速い順」つまり下から実行する**のが鉄則です。単体テストでバグが見つかれば、時間のかかるE2Eテストを動かす前に修正できるからです（Fail Fast）。

これからテストを書き始める初学者の方も、**単体テストから始めるのが圧倒的におすすめ**です。

理由は、単体テストが一番フィードバックが速く、エラーの原因特定が簡単だからです。まずは Vitest などで小さな関数のテストから成功体験を積んでいきましょう。

## 【言語別】定番テストツールはコレだ！

それでは、各言語で「これを選べば間違いない」という定番ツールを見ていきましょう。

### JavaScript / TypeScript (React, Vueなど)

- **単体テスト: `Vitest`**
  - Vite で作られたプロジェクトとの相性が抜群で、設定不要で爆速に動作します。React 専用ではなく、Vue や Svelte でも使えます。

- **E2Eテスト: `Playwright` vs `Cypress`**
  - **Playwright**: 複数ブラウザ対応など機能が豊富で、本格的なE2Eテストに向いています。
  - **Cypress**: GUIでのデバッグ体験が素晴らしく、初学者が始めやすいのが特徴です。

- **APIテスト: `supertest`**
  - Node.jsサーバーのAPIをテストする際の定番ライブラリです。

### Python

- **単体テスト: `pytest`**
  - 今のPython界隈では圧倒的なデファクトスタンダードです。
  - `assert 1 + 1 == 2` のように、シンプルな `assert` 文だけでテストが書ける手軽さと、豊富なプラグインが人気の理由です。

### Java

- **単体テスト: `JUnit 5`**
  - Java界の絶対王者であり、Spring Boot などのフレームワークでも標準採用されています。
  - 実際の開発では、オブジェクトを偽物に置き換える `Mockito` というライブラリとセットで使われるのが一般的です。

## まとめ

- テストは**「単体 → API → E2E」**のピラミッドで考える
- テストの実行も実装も、まずは**一番下の「単体テスト」から**始めるのが効率的
- 言語ごとの定番ツールを覚えよう！
  - **JS/TS**: Vitest
  - **Python**: pytest
  - **Java**: JUnit 5

この記事が、あなたのテストに対する苦手意識を少しでも解消できれば幸いです！
